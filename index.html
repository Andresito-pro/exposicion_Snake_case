¿Qué es PEP 8?

La Guía de Estilo Oficial

PEP 8 es el documento que proporciona las convenciones para escribir código Python. Su objetivo principal es mejorar la legibilidad y consistencia del código.

¿Qué herramientas verifican PEP 8?

En entornos profesionales, no lo revisamos a mano. Usamos linters y formateadores como:

  • Flake8: Analiza la sintaxis y estilo.
  • Black: Formateador automático (opinionated).
  • Pylint: Análisis profundo de calidad de código.

¿Cómo afecta PEP 8 a librerías externas?

Si usas una librería de terceros que usa otro estilo (ej. camelCase), debes mantener la convención de esa librería cuando interactúas con ella, pero usar PEP 8 estrictamente en tu lógica interna para mantener la coherencia de tu proyecto.

Variables y Funciones: snake_case

En Python, los nombres de funciones y variables deben ser en minúsculas, con las palabras separadas por guiones bajos para mejorar la legibilidad.

¿Por qué snake_case y no camelCase?

Python fue diseñado para ser altamente legible. Estudios en tipografía sugieren que el espacio (representado por el guion bajo) permite que el ojo humano identifique las palabras más rápido que el cambio de mayúsculas (camelCase), especialmente en variables largas. Además, hereda esta filosofía de C, el lenguaje en el que está implementado (CPython).

main.py 
def calcular_total(precio_base, impuestos):
    # Función usando snake_case
    total_final = precio_base + impuestos
    return total_final

Clases: PascalCase

El estándar dicta que los nombres de las clases deben usar UpperCamelCase (también conocido como PascalCase).

¿En qué casos se usa y por qué?

Se utiliza exclusivamente para la declaración de Clases (y tipos). La razón fundamental es la diferenciación visual: al ver una palabra que empieza con mayúscula en el código, el desarrollador sabe instantáneamente que se trata de un molde (Clase) y no de una instancia, variable o función. Previene colisiones de nombres.

models.py 
class UsuarioPremium:
    def __init__(self, nombre_usuario):
        self.nombre_usuario = nombre_usuario
        self.es_activo = True

Constantes: SCREAMING_SNAKE_CASE

Python no tiene verdaderas constantes a nivel de sistema. Se usa la convención de escribir la variable completamente en mayúsculas para indicar a otros programadores que ese valor no debe ser modificado.

# Esto es un aviso visual, no un bloqueo del sistema
MAX_INTENTOS = 3
API_KEY = "xyz_12345"
